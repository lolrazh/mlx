## Key Learnings
- **3B capacity ceiling at 91%.** Llama 3.2 3B with v4 data (1201 train) peaks at 91% — same data that gives Qwen3 4B 100%. The extra 1B params matter for quote-endquote scope and multi-word patterns.
- **V4 data gives consistent +4 pts across architectures.** Qwen3: 91%→100% (+9), Llama: 87%→91% (+4). Diminishing returns for smaller models.
- **Sixth confirmation: val loss is unreliable.** Iter 2000 (val 0.155) beat iter 700 (val 0.083) by 8 percentage points. Never use val loss as stopping criterion with 20 validation examples.
- **Adapter checkpoint benchmarking requires directory setup.** `run_benchmark.py --adapter-path` expects a directory with `adapter_config.json`, not a single safetensors file. Copy checkpoint + config to temp dir.

## Architecture Decisions
- **Skipped DWQ on Llama** — 91% bf16 doesn't justify the fuse+DWQ pipeline when we already have Qwen3 DWQ at 96%. User agreed.
- **8-bit QLoRA before Muon** — User suggested testing 8-bit base model to speed iteration. Smart variable isolation: test 8-bit precision separately, then use the winner for Muon experiments.
- **Qwen3 8-bit not Qwen3.5 8-bit** — User correctly identified that testing new model + new precision confounds two variables. Test 8-bit on known Qwen3 first.
