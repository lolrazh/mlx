## Technical Implementation

### The Core Insight: ASR Error Distribution
Whisper is trained on 680K+ hours. It knows **all common words** — Django, Postgres, YAML, Cloudflare, Seattle, Marriott, Xanax, Nocturne, etc. It does NOT misspell them. Users only spell things when:

1. **Names with no phoneme→spelling mapping** — Irish (Saoirse, Niamh, Siobhan, Aoife, Caoimhe, Eoin, Ciaran, Finbarr, Nuala, Roisin), Vietnamese (Nguyen), Chinese (Xiaoming, Xuhan), Turkish (Turan)
2. **Name homophones** — both spellings are real names (Sean/Shawn, Geoff/Jeff, Stephen/Steven, Marshall/Marshal, Allison/Alison, Chris/Kris)
3. **Brands = intentional misspellings of common words** — Wispr/Whisper, Lyft/Lift, Flickr/Flicker, Tumblr/Tumbler, Skool/School, Nite/Night, Tru/True

### Direction Matters
ASR outputs the **common** spelling. User spells to get the **uncommon** one:
- ASR: "DataDog" → User: "Spell that D-A-W-G" → Output: "DataDawg" ✅
- ~~ASR: "DataDawg" → User: "Spell that D-O-G" → Output: "DataDog"~~ ❌ (backwards)

## Bugs & Issues Encountered
1. **Fake ASR errors on common words** — Initial data had Whisper "misspelling" Django, Seattle, YAML, Postgres, Cloudflare, Nocturne, Xanax, Marriott, etc. These are all in Whisper's vocabulary.
   - **Fix:** Complete rewrite using only genuinely ambiguous names and brand homophones
2. **Reversed spell direction (DataDawg)** — Had ASR outputting the unusual spelling and user correcting to common
   - **Fix:** Flipped direction — ASR outputs common word, user spells the brand variant
3. **Zero-effect spell+caps examples** — 4 examples where input was already correctly spelled AND cased
   - **Fix:** Replaced with Irish/Chinese/Turkish names ASR genuinely mangles
4. **Approved bad data in first two review rounds** — Automated checks caught format issues but not semantic realism
   - **Fix:** User caught it. Lesson: format validation ≠ content validation

## Key Learnings
- **ASR error realism is non-negotiable** — The model will learn whatever patterns you train. Fake errors = fake corrections = production failures. Every input must be something Whisper would plausibly output.
- **Automated checks catch syntax, not semantics** — Bold detection, disfluency leaks, contamination checks all passed on garbage data. Need domain expertise (or at minimum, "would Whisper actually say this?") for content review.
- **Common-word spell corrections are red flags** — If the word is in any English dictionary, Whisper almost certainly knows it. Only names, brands, and domain jargon need spelling.
- **Direction of correction follows frequency** — ASR defaults to the more common spelling/word. User intervenes to get the less common one.
