## User Intention
User wanted to create a v3 training dataset that only includes categories with production Spoke triggers — eliminating 4 "dead" categories (formatting-xml, email, code-aware, hard-negatives) that would never reach the model at inference time. The goal was to stop wasting training capacity on patterns the model will never encounter in production, while strengthening the 9 remaining categories with fresh high-quality examples and rigorous multi-pass quality review.

- ✅ **Filtered v2 data into v3 structure** — Split formatting.json into caps/emphasis/at-symbol (dropped XML), filtered code-aware to camelcase only. 368 examples carried forward from v2.
- ✅ **Generated ~95 new examples via Kimi K2.5** — 6 parallel generation jobs across quote-endquote, caps, camelcase, emphasis, emoji, at-symbol. Custom prompts and validators for each new sub-category.
- ✅ **Multi-pass Sonnet review** — 5+ parallel review agents caught issues across at-symbol (17% fail rate!), quote-endquote, caps, emphasis, emoji, camelcase. All flagged examples fixed or deleted.
- ✅ **Standardized period-outside-quotes convention** — User prefers period always OUTSIDE closing quote mark. Applied uniformly across all 67 quote-endquote examples, updated validator + generation prompt.
- ✅ **Fixed 14 data quality failures from structural review** — 6 deleted (ambiguous/hallucinated), 8 corrected (word reorder, content drops, case errors).
- ✅ **Fixed emphasis format inconsistency** — One "Stress" trigger was producing bold instead of CAPS. Now 100% consistent: Emphasize/Bold → `**bold**`, Stress/Emphasis-on → CAPS.
- ✅ **Boosted weak categories to ~500 target** — Generated 65 more examples via Kimi, Sonnet review caught 10 more issues (7 deleted, 3 fixed), 58 kept.

- ✅ **Built leak-free train/valid/test splits** — merge.py deduplicates source pool against holdout sets. Zero leakage.
- ✅ **Created all v3 infrastructure** — split_v2.py, merge_sources.py, build_valid.py, boost.py, merge.py

## Bugs & Issues Encountered
1. **Kimi pipeline overwrote v2 files** — generate.py saves to `spoke/data/final/`, clobbering v2 data
   - **Fix:** Saved Kimi output to `v3/kimi/`, restored v2 with `git checkout`, built `merge_sources.py` to combine
2. **At-symbol category had 17% failure rate** — LLM-generated examples kept rewriting word order, dropping content, hallucinating words
   - **Fix:** Deleted worst offenders, fixed correctable ones. At-symbol is the hardest category for synthetic data because the "surgical insertion" pattern conflicts with LLMs' desire to "improve" text.
3. **validate() API mismatch** — boost.py called `validate(category, input, ideal)` but signature is `validate(category, data_list)`
   - **Fix:** Used `VALIDATORS[category](pair)` directly instead
4. **Period placement convention flip-flop** — Initially "fixed" test ID 9 from period-outside to period-inside, then user clarified preference is always-outside
   - **Fix:** Reverted, standardized all 67 quote-endquote examples to period-outside, simplified prompt to one rule
5. **Heuristic category detection failed for valid set filtering** — Email examples with "wait no" triggered self-correction detector
   - **Fix:** Replaced heuristic detection with explicit line-number filtering

## Key Learnings
- **At-symbol is the synthetic data trap** — LLMs that generate training data want to "improve" text, but ASR post-processing needs surgical edits. This category consistently had the worst quality.
- **"uh" in emphasis creates a system-prompt vs user-directive conflict** — If the system prompt says "remove uh" but the user says "stress uh", the data teaches the model that user overrides system rules. Resolved by always letting system-prompt filler removal win.
- **Period placement needs ONE rule, not conditional logic** — The v2 convention had inside/outside depending on speaker intent. Simplified to always-outside — one rule, no exceptions.
- **Emphasis trigger-to-format mapping must be 1:1** — "Emphasize"/"Bold" → `**bold**`, "Stress"/"Emphasis on" → ALL CAPS. Even one outlier (1/27) creates ambiguity.
- **Dedup at merge time, not generation time** — Source pools share examples with test/valid sets. merge.py catches these at the final stage, ensuring zero leakage.

## Architecture Decisions
- **v3/source/ as single source of truth** — All category data lives in source JSON files. merge.py reads these + holdout sets and produces the final JSONL splits. No manual JSONL editing.
- **Separate kimi/ and kimi-boost/ directories** — Preserves raw generation output for traceability. Source files contain the merged+reviewed versions.
- **boost.py bypasses the main pipeline** — Avoids the `final/` directory overwrite issue. Reads from v3/source/ for dedup, writes to v3/kimi-boost/.
- **test_set.json stays in spoke/bench/** — The benchmark script reads from there. test.jsonl in v3/ is a convenience copy in chat format.
