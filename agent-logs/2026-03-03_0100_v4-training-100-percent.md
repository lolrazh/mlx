# V4 Data Training: Qwen3 T2 Hits 100% Accuracy

**Date:** 2026-03-03
**Agent:** Claude Opus 4.6
**Status:** ✅ Completed

## User Intention
User wanted to train a new Qwen3 4B model (T2) on the expanded v4 dataset (1201 examples, up from 535 in v3) and benchmark it against the previous best run (T11-ext, 91%). The goal was to see if more diverse data — including 289 hard negatives — could break the 91% accuracy ceiling that all 4B-class models had converged to on v3 data.

## What We Accomplished
- ✅ **Merged v4 dataset** - Combined v3 train (535) + new regular (377) + hard negatives (289) = 1201 shuffled training examples
- ✅ **Trained Qwen3 T2** - 2000 iters on v4 data, Qwen3-4B-Instruct bf16, all 36 layers, ~3.5 hours
- ✅ **100% benchmark accuracy** - 23/23 exact matches on v3 test set (up from T11-ext's 91%)
- ✅ **Validated both checkpoints** - Iter 1100 (best val loss 0.065) and iter 2000 (final, val loss 0.091) both scored 100%
- ✅ **Ad-hoc generalization tests** - Model correctly handled novel spell-replacements (Celero→Silero, Gamma→Gemma), emoji, and self-correction on inputs NOT in the training set
- ⚠️ **Wispr edge case** - Model drops "Flow" from "Whisper Flow" on multi-word product name spell-replace — scopes replacement too broadly

   - **Fix:** Used actual file contents, total became 1201 instead of planned 1179
2. **Peak memory higher than expected** - 18.6 GB vs expected ~9 GB for Qwen3 without grad_checkpoint
   - **Note:** Larger dataset creates bigger computation graphs. Still within 24 GB budget.
3. **Val loss overfitting after iter ~1100** - Val loss rose from 0.065 to 0.091 while train loss hit 0.001
   - **Resolution:** Both checkpoints still scored 100% accuracy — val loss is not a reliable proxy for accuracy with only 20 val examples

## Key Learnings
- **Val loss ≠ accuracy** with small validation sets. 0.065 vs 0.091 val loss both yielded identical 100% accuracy. The model becoming "more confident" raises cross-entropy loss but doesn't change outputs.
- **More diverse data > more epochs on same data.** v3 (535 examples) plateaued all 4B models at 87-91%. v4 (1201 examples with hard negatives) broke through to 100%.
- **Hard negatives teach restraint.** The model correctly handles "don't change" cases and scopes edits tightly (Celero→Silero without touching "VAD").
- **Iter 2000 was faster at inference** (1.82s avg) than iter 1100 (2.24s avg) despite higher val loss — more confident model generates tokens more decisively.
- **User's intuition was right** — iter 2000 performed equally well despite worse val loss, matching prior experience with T11-ext.

## Architecture Decisions
- **Single-phase training** (shuffle everything together) rather than curriculum learning — simpler and effective at this scale
- **Kept val/test sets identical to v3** — sacrificed opportunity to expand evaluation for direct comparability with all prior runs
- **Used iter 2000 as primary checkpoint** — both work, but 2000 is faster at inference and represents the "final" state
